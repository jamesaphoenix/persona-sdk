Getting Started

Getting Started

This page will walk you through the basics of creating personas, using distributions, and optimizing prompts. For complete documentation, refer to the API Reference.

Installation

npm install @jamesaphoenix/persona-sdk
# or
yarn add @jamesaphoenix/persona-sdk
# or
pnpm add @jamesaphoenix/persona-sdk

Defining a persona

Before you can do anything else, you need to create a persona. For this guide, we'll use the PersonaBuilder.

import { PersonaBuilder, PersonaGroup, NormalDistribution } from '@jamesaphoenix/persona-sdk';
 
// Create a single persona
const persona = PersonaBuilder.create()
  .withName('Alice Johnson')
  .withAge(28)
  .withOccupation('Software Engineer')
  .withSex('female')
  .withAttribute('yearsExperience', 5)
  .build();
 
// Create a group with statistical distributions
const group = new PersonaGroup('Tech Workers');
group.generateFromDistributions(100, {
  age: new NormalDistribution(32, 5),
  occupation: 'Developer',
  sex: 'other',
  yearsExperience: new UniformDistribution(1, 10)
});

Using statistical distributions

Create realistic personas at scale using statistical distributions:

import { PersonaGroup, NormalDistribution, UniformDistribution } from '@jamesaphoenix/persona-sdk';
 
const group = new PersonaGroup('Tech Workers');
group.generateFromDistributions(100, {
  age: new NormalDistribution(32, 5),
  occupation: 'Developer',
  sex: 'other',
  yearsExperience: new UniformDistribution(1, 10)
});

The SDK includes Normal, Uniform, Exponential, Beta, and Categorical distributions.

Optimizing prompts

import { 
  BootstrapOptimizer, 
  ExactMatch,
  MockModule,
  createMockLanguageModel 
} from '@jamesaphoenix/persona-sdk';
 
// Create a module to optimize
const module = new MockModule('Generate a persona for: ');
 
// Create training data
const trainData = [
  { input: 'tech startup', output: 'Young professional, tech-savvy' },
  { input: 'healthcare', output: 'Medical professional, caring' }
];
 
// Optimize the prompt
const optimizer = new BootstrapOptimizer({
  maxLabeled: 5,
  metric: ExactMatch,
  teacherModel: createMockLanguageModel()
});
 
const result = await optimizer.optimize(module, trainData);
console.log(`Optimized score: ${result.finalScore}`);

Handling errors

When creating personas fails, the builder will throw descriptive errors:

try {
  const persona = PersonaBuilder.create().build();
} catch (error) {
  console.error(error);
  // Error: Persona name is required
}

For safer persona creation without exceptions, validate first:

const builder = PersonaBuilder.create();
if (!builder.name || !builder.age) {
  // Handle missing required fields
}

Inferring types

Persona SDK provides full TypeScript support with inferred types:

import { Persona, PersonaAttributes } from '@jamesaphoenix/persona-sdk';
 
// Define custom attributes
interface CustomAttributes extends PersonaAttributes {
  department: string;
  skills: string[];
  remote: boolean;
}
 
// Use with PersonaBuilder
const developer: Persona<CustomAttributes> = PersonaBuilder.create()
  .withName('Alex')
  .withAge(28)
  .withOccupation('Developer')
  .withSex('other')
  .withAttribute('department', 'Engineering')
  .withAttribute('skills', ['TypeScript', 'React'])
  .withAttribute('remote', true)
  .build();
 
// TypeScript knows about your custom attributes
console.log(developer.attributes.skills); // string[]
